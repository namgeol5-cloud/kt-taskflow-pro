# 01-product — 제품 정의

## 제품 목표

팀 업무를 시각화한다. **"지금 누가 뭐 해?"** 라는 질문이 사라지게 만드는 것이 목표다.

---

## 페르소나

| 항목 | 내용 |
|---|---|
| 대상 | 10명 규모 스타트업 팀 리더 |
| 연령대 | 30~40대 |
| 상황 | 슬랙·구두 보고 없이도 팀 진행 상황을 한눈에 파악하고 싶다 |
| 불편함 | 업무 현황을 확인하려면 매번 팀원에게 물어봐야 한다 |

---

## MVP 범위

### CRUD 4종 — 모든 동작이 화면에서 이루어진다

| 동작 | 설명 |
|---|---|
| 추가 (Create) | 폼에서 직접 업무 생성 |
| 목록 (Read) | 전체 업무를 카드·리스트로 표시 |
| 수정 (Update) | 인라인 또는 모달에서 편집 |
| 삭제 (Delete) | 확인 후 제거 |

### 업무 속성

- **상태 분류**: `todo` / `in_progress` / `done`
- **마감 시각 (`due_at`)**: 날짜가 아닌 **시간까지** 지정하고 표시
  - 형식 예시: `2026-05-12 18:00`
  - 저장 형식: ISO 8601 (`2026-05-12T18:00:00`)

### 테마

- 라이트 / 다크 테마 토글
- 선택값은 `localStorage`에 저장 → 새로고침 후에도 유지

### 반응형

- 모바일 최소 지원 너비: **360px**
- 360px에서 레이아웃이 깨지지 않는 것이 성공 기준

## UI 톤 — macOS 스타일

| 요소 | 규칙 |
|---|---|
| 모서리 | 둥근 모서리 (`border-radius: 12px` 이상) |
| 그림자 | 부드러운 그림자 (`box-shadow` 저채도·저불투명도) |
| 카드 | 반투명 카드 (라이트: `rgba(255,255,255,0.7)` / 다크: `rgba(30,30,30,0.7)`) |
| 폰트 | 시스템 폰트 (`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`) |

## 확장 계획 (MVP 이후)

우선순위 순서대로 나열한다.

1. JWT 로그인 · 회원가입
2. 팀(조직) 단위 업무 분리
3. Kanban 보드 뷰
4. 팀 채팅
5. CI/CD 파이프라인 자동화

---

## MVP 범위 외 (보류)

| 항목 | 사유 |
|---|---|
| 외부 결제 | MVP 단계 불필요 |
| 네이티브 앱 | 웹 반응형으로 대체 |
| WebRTC | 복잡도 과다 |
| 외부 캘린더 연동 | API 의존성 증가 |
| 파일 업로드 | 스토리지 설계 별도 필요 |
| WebSocket 실시간 | MVP는 폴링으로 대체, 확장 단계에서 도입 |

---

## 성공 기준

MVP가 완료됐다고 선언하려면 아래 항목을 **모두** 충족해야 한다.

| 기준 | 조건 |
|---|---|
| 데이터 유지 | 새로고침 후에도 업무 목록이 그대로 남는다 |
| 반응형 | 360px 뷰포트에서 레이아웃이 깨지지 않는다 |
| API 응답 | 모든 엔드포인트 응답 시간 200ms 이하 |
| CRUD 동작 | 추가·목록·수정·삭제 4종이 화면에서 모두 동작한다 |
| 테마 토글 | 라이트/다크 전환이 작동하고 새로고침 후에도 유지된다 |
