# 03-design — 기술·설계 결정

## 의존성 추가 정책

새 패키지·라이브러리를 도입하기 전, **이 문서에 결정 사유를 먼저 기록**해야 한다.
사유 없이 `package.json` / `requirements.txt` 를 변경하는 것은 절대 규칙 위반이다.

---

## 설계 결정 8선

### 1. 백엔드 프레임워크

| 항목 | 내용 |
|---|---|
| **선택** | FastAPI |
| **대안** | Django, Express |
| **근거** | 타입 힌트 기반 자동 스키마, 비동기 지원, 경량 구조로 소규모 MVP에 최적 |
| **트레이드오프** | Django 대비 ORM·어드민·배터리 포함 기능 없음 → 직접 설정 비용 발생 |

---

### 2. 프론트엔드

| 항목 | 내용 |
|---|---|
| **선택** | Vanilla JS + Tailwind CDN |
| **대안** | React, Vue |
| **근거** | 빌드 도구 없이 즉시 실행, CDN 한 줄로 Tailwind 적용, 번들러 설정 비용 제로 |
| **트레이드오프** | 컴포넌트 재사용·상태 관리가 수동이라 규모 커지면 유지보수 난이도 상승 |

---

### 3. 데이터베이스

| 항목 | 내용 |
|---|---|
| **선택** | SQLite (개발) → PostgreSQL (운영) + SQLAlchemy ORM |
| **대안** | MySQL, MongoDB |
| **근거** | SQLite는 설정 없이 파일 하나로 즉시 시작, SQLAlchemy로 DB 교체 시 코드 변경 최소화 |
| **트레이드오프** | SQLite는 동시 쓰기 성능 낮음 → 운영 전 PostgreSQL 전환 필수 |

---

### 4. CSS 방법론

| 항목 | 내용 |
|---|---|
| **선택** | Tailwind CSS 단독 사용 |
| **대안** | styled-components, CSS Modules |
| **근거** | 유틸리티 클래스로 일관성 확보, JS 없이 스타일 작성, 빌드 없이 CDN 사용 가능 |
| **트레이드오프** | HTML 클래스 문자열이 길어짐 → styled-components 혼용 금지로 일관성 유지 |

---

### 5. 실시간 업데이트

| 항목 | 내용 |
|---|---|
| **선택** | MVP: 3초 폴링 (`setInterval`) |
| **대안** | WebSocket, SSE |
| **근거** | 구현 복잡도 최소화, 10명 규모 팀에서 폴링 부하 허용 범위 내 |
| **트레이드오프** | 3초 지연 존재, 서버 요청 주기적 발생 → WebSocket은 확장 단계에서 도입 |

---

### 6. 상태 관리

| 항목 | 내용 |
|---|---|
| **선택** | 모듈 변수 + DOM 직접 갱신 |
| **대안** | Redux, Zustand, Pinia |
| **근거** | Vanilla JS 환경에서 외부 라이브러리 없이 단순 상태 유지 가능 |
| **트레이드오프** | 상태가 분산되면 버그 추적 어려움 → 상태 변수를 단일 모듈(`state.js`)에 집중 관리 |

---

### 7. 디자인 시스템

| 항목 | 내용 |
|---|---|
| **선택** | macOS UI 톤 자체 구현 |
| **대안** | Material Design, Ant Design |
| **근거** | 외부 컴포넌트 라이브러리 없이 Tailwind 토큰만으로 일관된 macOS 감성 구현 |
| **트레이드오프** | 컴포넌트를 직접 만들어야 하므로 초기 UI 작업량 증가 |

#### 디자인 토큰 (Tailwind 클래스 기준)

| 토큰 | Tailwind 클래스 | 용도 |
|---|---|---|
| 둥근 모서리 | `rounded-xl` | 카드·버튼·모달 |
| 그림자 | `shadow-lg` | 카드 깊이감 |
| 반투명 배경 | `backdrop-blur-md` | 카드·모달 배경 |
| 폰트 | `font-sans` + CSS 시스템폰트 | 전체 텍스트 |
| 터치 타깃 | `min-h-[44px] min-w-[44px]` | 버튼·아이콘 (모바일 접근성) |

### 8. 테마 (라이트 / 다크)

| 항목 | 내용 |
|---|---|
| **선택** | Tailwind `dark:` 변형 + `localStorage('theme')` |
| **대안** | CSS 변수 수동 전환, OS 테마만 따르기 |
| **근거** | Tailwind `dark:` 클래스로 별도 CSS 없이 다크 스타일 선언, `localStorage`로 사용자 선택 유지 |
| **트레이드오프** | `class="dark"` 를 `<html>` 에 수동 토글해야 함 → 초기 로드 시 깜빡임(FOUC) 방지 스크립트 필요 |

#### 초기화 로직

```
1. localStorage('theme') 값 확인
2. 없으면 prefers-color-scheme 미디어쿼리 확인
3. 결과에 따라 <html class="dark"> 추가 또는 제거
4. 토글 버튼 클릭 시 class 전환 + localStorage 갱신
```
